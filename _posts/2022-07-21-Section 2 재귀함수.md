## 간단정리
 재귀호출은 메서드가 메서드 자신을 부르는 것이다.   
따라서 중첩되면서 기능을 구현하게 된다.   
하지만 중첩된다는 것은 흐름을 직관적으로 파악하기 힘들다는 단점을 불러온다.   
따라서 이 흐름을 잘 파악하도록 하자.

내가 재귀호출을 사용하는 경우라면 입력값과 출력값을 잘 구분하자.   
그리고 문제를 쪼개보자   
쉽게 할 수 있는 부분 (더 이상 쪼개기 불가)    
쉽게 할 수 있는 부분 (재귀적으로 불러오는 부분)    
으로 나누어 접근해보자.  

한편 재귀호출은 많은 메모리 공간을 차지하게 된다. 그렇기 때문에 현업에서 사용하게 될 기회는 매우 적을 것이다. 하지만 느껴지는게.. 결국 백엔드는 효율성을 극대화 하는 것이 좋을 것이고, 이를 위해서는 더욱 더 효율적인 코드를 작성하기 위해 알고리즘적 사고에 익숙해져야 할 것이다. 따라서 재귀 자체는 메모리 관점에서는 효율적인 코드 작성이 아니지만, 작성하는데 들이는 내 사고력을 강화하는데 도움이 된다. 이건 무조건이다. 소홀히 하지 말자.

### 1. recursion(재귀)
    메서드가 메서드 자신을 호출
    
자신을 호출한다??   
메서드안에서 다른 메서드를 호출하는 것과 다를 바 없다.
메서드는 메모리에 저장되어 있고, 그저 그 공간에서 메서드를 불러오는 것 뿐이기 때문이다.

```java
public int add(x, y){
    x = x+y;
    System.out.println(x);
    add(x,y);
}
```
이렇게 되면 계속 x에 y값이 누적되어 더해질 것이다.
이 때의 add를 재귀호출이라고 하겠다.
위 코드는 예시이지만 재귀함수는 '탈출'할 때가 언제인지 생각한 후 '탈출 코드'를 작성해 주어야 한다.

#### a. 장점
- 코드가 간결해지고 수정이 용이해진다.
    - 재귀는 결국 똑같은 동작을 반복한다. 위 add(x,y)를 반복문으로 했으면 훨씬 길어지고 누적될수록 복잡해졌을것

#### b. 단점
- 메모리를 많이 사용한다.
    - 반복적으로 메서드를 호출하는 방식이기 때문에 지역변수들과 반환값을 모두 process stack에 저장한다.
- 코드의 흐름이 직관적으로 보이지는 않는다.

### 3. 코드 작성해보기

#### a. 문제
    배열의 요소들을 곱해라

#### b. 반복문으로
```java
public int mutiple(int[] arr){
    int result = 1;
    for(int i = 0; i < arr. length ; i++){
        result = result * arr[i];
    }
    return result;
}
```

#### c. 재귀호출로
```java
public int multiple(int[] arr){
    if(arr.length == 0) return 1;

    int head = arr[0];
    int[] tail = Arrays.copyOfRange(arr,1,arr.length-1);

    return head * multiple(tail);  // 재귀호출로 tail의 길이가 0이 될때까지 head를 생성할것
}
```

#### d. 재귀적 사고 연습

##### 1.입력값과 출력값
    위 문제의 입력값은 배열이고,
            출력값은 int형 정수 값이 된다.

##### 2.문제 쪼개고 경우의 수 나누기
    [쪼갤 수 있는 경우]
    배열의 요소들을 하나씩 정수형으로 꺼내서 곱해주면 된다.
    [쪼갤 수 없는 경우]
    배열에서 요소들을 꺼내고 배열에 아무것도 들어있지 않으면 1로 리턴된다. (곱해도 값에 영향을 주지 않는다.)

##### 3.단순 문제 해결
    쪼갤수 없는 경우가 쉽겠다.
    if(arr.length == 0) return 1;

##### 4.복잡 문제 해결
    배열에서 수를 계속 꺼내는 경우
    int head = arr[0];
    int[] tail = Arrays.copyOfRange(arr,1,arr.length-1);
