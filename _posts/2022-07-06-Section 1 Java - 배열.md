- 페어분의 학습자세가 굉장히 인상깊었다. 유튜브를 진행하던 당시 메타인지에 굉장히 흥미를 느꼈었고, 내가 아는 것과 모르는 것을 구분해오고자 꾸준한 생각 너머의 생각을 하는 습관을 들여왔었다. 그리고 이번에 유튜브 파트너 친구 이후로 타인으로부터 오랜만에 그런 부분들을 느꼈다. 왜 안되는지에 대한 생각을 나는 무엇을 모르고 무엇을 알고 있다' 로 부터 시작하니까, 또 그런 부분들을 동시에 공감하고 있으니까, 페어 진행이 굉장히 부드러웠고 동시에 속도도 빨랐으며, 심화학습도 가능하지 않았나 싶다.

## 정리
배열은 왜 필요할까???

    선생님이 되는 꿈을 꿨다. 꿈 속 세상은 중간고사를 막 끝내고 나는 점수정리 지옥에 빠져있다.

이 경우 학생 하나 하나 변수를 선언해서 값을 할당하면 코드는 엄청 늘어나고, 가독성이 떨어질 것이다. 그리고 관리도 힘들겠지

따라서 배열을 사용한다.

**자료들을 모은다는 관점은 더 확대될 수 있다. 추후 공부하겠지만    
자료형 -> 배열 -> 컬렉션과 같은 구조체 -> 객체 의 순서로 구분할 수 있겠다. 
### 1. 1차원 배열

```java
자료형 변수명 = new 자료형[배열크기];

int score = new int[30];
//이렇게 하면 int형 변수 score로 30개의 리터럴을 관리하게 된다.

int score = new int[] {1,2,3};
int score = {1,3,6,7,8,9}; // 이런 경우들도 가능하다

``` 

배열은 방법론만큼 중요한게 '어떤 타입인가' 라고 생각한다. 지금까지 자바에서 제공하는 기본형 타입들은 변수에 리터럴을 저장한다. 하지만 배열은 ***리터럴이 저장되어 있는 곳의 시작 메모리 주소***를 저장한다.

이 이유는 ***선언할때는, '할당할 때' 저장할 배열의 크기가 몇인지 예측할 수 없기 때문이다.*** 물론 바로 할당을 해주면 한줄에서 알 수 있는거 아니야?? 라고 말할 수 있겠지만 그건 사람입장에서나 그렇지 컴퓨터 입장에서는 아니다.

### 2. 2차원 배열
이차원 배열은 차원이 하나가 늘어났을 뿐 1차원 배열과 크게 다르지 않다.

```java
int[][] list = new int[3][3]; //첫 대괄호는 행의 갯수 두번째 대괄호는 열의 갯수
                              //첫 대괄호는 외부배열, 두번재 대괄호는 내부배열
int[][] list = new int[][]{
    {1,2,3},{4,5,6},{7,8,9}
};
``` 

하지만 차이라면

마지막 차수의 경우는 값을 지정해주지 않아도 되는 가변배열로 할당할 수 있다는 것이다.

```java
int[][] ages = new int[5][];

ages[0] = new int[5];
ages[1] = new int[6];
ages[2] = new int[7];
ages[3] = new int[8];  
ages[4] = new int[9];
```
