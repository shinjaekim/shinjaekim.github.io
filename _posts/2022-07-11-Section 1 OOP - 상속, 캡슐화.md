## 간단정리

- 클래스의 멤버들을 하위 클래스들이 상속받아 공유함으로서 **코드의 중복을 막고 재사용**한다.
- 포함 또한 클래스의 멤버를 재사용한다는 측면에서 같다
    - 구분은 이렇게하자. **~이다 -> 상속 / ~를 가지고 있다 -> 포함**
- 메서드를 **오버라이딩** 하는 것이 가능하다.
    -오버라이딩 후 상위 클래스의 메서드를 부르고자 한다면 super를 이용하자
- JAVA는 **단일 상속**만 허용한다.
- 모든 클래스는 Object클래스의 자손이다.
- 상속은 **다형성** 으로 이어진다.

- 캡슐화는 **정보를 은닉**하고 **불필요한 노출을 방지**한다.
- 패키지는 관계있는 클래스들의 묶음으로 하나의 디렉토리이다.
- 패키지선언을 가장 위에, 패키지 import을 2순위로 한다.
- 접근제어자와 **게터 세터**를 이해하도록 하자.

### 1. 상속

    - 기존의 클래스를 재사용 하여 새로운 클래스를 작성한다.
    - 확장되었다는 표현이 더 맞겠다. → extends
    - 메서드를 오버라이딩 하여 일일이 객체를 생성하는 경우를 방지할 수 있다.
    - 단 하나의 상위 클래스를 가질 수 있다.

<BR>

#### a. 장점
- 겹치는 속성을 계속 안 써도 되니까 중복이 줄어든다.
    - 하위클래스에서 상위클래스에게 상속받은 것은 안써도 된다는 것.

- 기존의 클래스를 재사용 하여 새로운 클래스를 작성한다.

> **중복을 제거하여 작성 코드의 줄수를 줄일 수 있을 것이다.** 또한 상위클래스를 선언, 하위클래스로 인스턴스 생성을 가능하게하는 **다형적 표현이 가능**하다.
> 
> - *적은양의 코드를 작성할 수 있음은 무엇을 뜻하는가?? static의 경우에도 공통된 변수를 사용 가능했고… 내부클래스 또한 여러가지 활용이 가능했다..*

#### b. 포함
- 클래스의 멤버 로 다른 클래스 타입의 참조변수를 선언하는 것
- 상속과의 구분은..
    - ~은 ~이다 - 상속
    - ~은 ~을 가지고 있다 - 포함

예시) - 이해를 위해 복붙

```java
public class Employee {
    int id;
    String name;
    Address address;

    public Employee(int id, String name, Address address) {
        this.id = id;
        this.name = name;
        this.address = address;
    }

    void showInfo() {
        System.out.println(id + " " + name);
        System.out.println(address.city+ " " + address.country);
    }

    public static void main(String[] args) {
        Address address1 = new Address("서울", "한국");
        Address address2 = new Address("도쿄", "일본");

        Employee e = new Employee(1, "김코딩", address1);
        Employee e2 = new Employee(2, "박해커", address2);

        e.showInfo();
        e2.showInfo();
    }
}

class Address {
    String city, country;

    public Address(String city, String country) {
        this.city = city;
        this.country = country;
    }
}
// 
```

#### c. 오버라이딩

메서드 오버로딩은 같은 이름의 메서드, 다른 매개변수를 이용한 작성 기법이었다. 같은 기능을 하지만 매개변수가 다른경우를 위한 설계였다. **오버로딩은 같은 클래스 내에서의 작성기법이었다면 오버라이딩은 상위, 하위 클래스 관계에서의 작성 기법이다. 오버라이딩을 진행할 시에 하위클래스의 메서드가 작동하게 된다.**
    조건
        메서드의 선언부가 상위클래스와 완전히 일치해야한다.
            오버로딩은(동일 클래스 내에서 작성) 선언부의 매개변수는 달라야 했다.
        접근 제어자의 범위가 상위 클래스의 메서드보다 같거나 넓어야 한다.
        예외는 상위 클래스의 메서드보다 많이 선언할 수 없다.
        
 
특히나 다형성을 이용할 수 있다. **상위 클래스를 선언한 후 하위클래스를 인스턴스화** 하면 오버라이딩 한 메서드를 이용 가능하며, 관리도 편리할 것이다.

#### d. super() 과 super.

this.와 this()를 떠올려보자. **this. 은 인스턴스 자체**를 가리켰다. 생성자의 매개변수가 클래스의 인스턴스 변수와 이름이 같은 경우 구별을 할때 인스턴스 이름 앞에 this.를 붙여줬다. **this()의 경우에는 생성자에서 다른 생성자를 호출할때** 쓰였다. 그렇다면 super()와 super.은??? 똑같이 이해하자. **super.은 바로 위의 상위 인스턴스**를 가리킨다. 그리고 **super()은 상위 클래스의 생성자**를 가리킨다.
 

예시) super()

```java

		yellowcar vehicle = new yellocar();

class VividCar{
```

### 2. 캡슐화

#### a. 장점
객체를 보면 객체는 그와 관련된 속성과 기능의 집합이다. 이렇게 관련된 데이터와 메서드들을 모아서 하나의 캡슐로 만드는 것. 그것이 캡슐화이다. 이렇게 되면 **데이터를 보호하고, 내부적으로만 사용되는 데이터의 불필요한 외부 노출을 방지한다.**

**데이터를 보호한다는 것을 생각해보자.** 
    한 객체 안에 멤버들을 외부에서 직접접근이 불가하게 제어자를 설정해주고 getter와 setter를 이용해보자. 데이터가 의도하지 않은 방향으로 변화되는 것을 방지할 수 있을 것이다.

**불필요한 외부 노출을 방지한다는 것을 생각해보자.** 
    컴파일러는 main메서드를 읽는다. 즉 실질적인 사건들을 - 객체들간의 관계를 - main에 작성할 것이다. 이 때 클래스들로부터 인스턴스를 생성해서 관계를 설정하면 외부사람들은 데이터를 보려면 클래스를 일일이 찾아야한다. 이렇게 불필요한 노출을 피할 수 있다.

이런 장점들은 **정보 은닉**으로서도 표현 할 수 있겠다.
<Br>

#### b. 패키지
클래스는 멤버들의 캡슐화를 달성한다. 그렇다면 패키지는 **클래스와 인터페이스의 캡슐화**를 달성한다.
이는 이렇게 생각할 수도 있다.
    클래스 파일은 무조건 하나의 패키지 디렉토리에 속한다.

이러한 디렉토리는 . 으로 구별하게 된다.
    java.lang.String~~

##### 1.패키지의 선언
패키지의 선언은 주석을 제외하고 소스코드의 가장 위에 위치한다.
    package 패키지 네임
**class이름은 대문자로 시작** 하던 것을 기억하자. **package는 헷갈리지 않도록 소문자**로 쓰는 편이다.

##### 2.패키지의 장점
쉽다. 클래스의 이름이 다르면 각각의 클래스 간에는 같은 이름의 메서드, 변수들을 선언해도 충돌이 일어나지 않는다. 마찬가지로 패키지로 구분하면 **패키지별로 동일안 이름의 클래스들이 존재하는 것이 가능**하다.

##### 3.import
소스코드의 첫줄에 package선언이 있다면 그 다음은 import이다. 이것이 무어냐. 타 패키지의 클래스들을 사용할 수 있도록 선언해 주는 것이다.(정확히는 매번 패키지 이름을 치지 않고...) 앞서 언급한 . 을 통해 접근할 수도 있고, 아예 한 디렉토리 아래에서 *을 이용하여 다 불러올 수도 있다.
    import java.lang.String~~
    import java.lang.*
이때 ***에 낚이지 말자.**
하위 디렉토리의 모든 클래스를 불러오는 것은 아니다. 단지 *이 표시된 동등한 레벨의 디렉토리에서의 클래스들을 불러온다.

#### d. getter, setter
캡슐화의 핵심이지 않나 싶다.
외부에서는 데이터에 직접 접근을 불허하고, 메서드를 통해서 호출한다.

```java
class Do{
    private int a;
    private int b;

    public int getA() {  // getter는 단축기 생성도 가능하지민
        return a;        // 반환타입이 변수와 같다는 것을 생각하자.
    }
    public int getB() {
        return b;
    }

    public void setA(int a) {  //setter는 매개변수로 멤버변수를 받고
        this.a = a;            //this를 이용하는구나
    }
    public void setB(int b) {
        this.b = b;
    }
}
```
