## 정리


### 1. LINUX

    - CLI - command line interface
        - 인터페이스 - 운영체제 조작
        - GUI - graphic user interface : 지금까지 나의 컴퓨터 사용 법

(220819)
현재 내가 이 마크다운 언어를 작성하고 있는 VScode, 인텔리제이 그리고 지금까지 써왔던 무수히 많은 프로그램들은 GUI(Graphic User Interface)이다. 하지만 내가 이 글을 수정하고 git에 commit할 때 사용하는 터미널은 CLI(Command Line Interface)이다. 한 작업을 a라고 가정했을때 GUI에 비하여 그래픽 요소를 사용하지 않기 때문에, 즉, 하나의 작업을 하는데 가용해야하는 자원이 더 적기때문에 ***더 안정적이며, 속도가 빠르다*** 물론 제어하기에 더 직관적인 것은 GUI일 것이다.

하지만 CLI를 사용하도록 유혹하는 요소가 단순히 빠르고 안정적이다 이 두가지뿐이라면 차라리 난 GUI를 쓰겠다. 하드웨어의 발전으로 내가 일반적으로 컴퓨터를 할 때 사용하는 프로그램들 사이에서는 유의미한 차이를 만들어내지 않을 것이라 생각하기 때문이다.

다만, 일반인이 아니라 백엔드, 그리고 서버를 배우고 있는 사람으로서는 CLI를 익혀야 한다. 내가 관리하는 서버컴퓨터의 수가 많아진다면?? 당연히 그 수만큼의 입출력장치(모니터, 마우스, 키보드)를 부착하는 것이 가능할까?? 컴퓨터 한대 한대를 조작하기 위해서?? CLI를 통해 서버 컴퓨터로 접속을 하면 여러대의 서버 컴퓨터를 활용할 수 있음에도??

CLI가 낫다 GUI가 낫다를 논하는 것이 아니다. 일단은 이렇게 생각하고 접근하자.

> CLI는 컴퓨터와 소통하는 가장 기본적인 방법이다.

    - 터미널을 통하여 기존 GUI로 하던 것 다 할 수 있다.
- 서버에 쓰이는 운영체제이다
    - 예로 아마존의 서버 클라우드(aws) ec2에 터미널을 통하여 쉽게 접근 가능
    - 개인 pc (CLI 구동) → 서버에 접근
    
<br>

> (왜 서버에 쓰일까 생각을 해봤다. 이건 나만의 생각이다. 과거에는 개인용 pc가 상용화 되어 있지 않았을 뿐더라 성능이 좋지 못했다. 따라서 하나의 pc를 여러명이 썼는데, 그당시 쓰던 것이 리눅스였지 않을까, 또한 그러한 상황과 컴퓨터 개발 초기 서버를 구축하던 상황이 맞물려 리눅스는 오픈소스로 동시작업이 편리한 운영체제로 굳혀지지 않았을까)

- 코드는 디렉토리(폴더)에 묶음 패키지로 있다. 이를 구동시키기 위해서 리눅스를 공부하는 것이다.
- CLI의 특징은 GUI와 달리 그래픽에 자원을 소모하지 않기 때문에 빠르게 동작한다.


#### b. 명령어
    코드 스테이츠 자료로 알게 된 것
        pwd
            - print working directory
            - 현재 폴더(경로)를 보여준다
        ls
            - list
            - 현재 directory안에 있는 폴더와 파일 리스트를 보여준다
            - -l → 속성?들을 보여준다.
            - -a → 숨겨진 모든 폴더와 파일들을 보여준다.
        mkdir
            - make directory
            - 폴더를 만든다.
                - 같은 이름이 있으면 만들지 않는다.
        cd
            - change directory
            - 선택한 폴더로 이동한다.
        touch
            - 파일을 생성한다.
            - 어원을 생각해보자. 만약 이미 생성된 이름의 파일을 touch로 생성하면 최근 작성일자가 업데이트된다.
        >
            - 실행결과를 파일로 저장한다.
            - ls > lslist.txt -> 실행하면 lilist의 파일안에 ls의 실행결과가 저장되어 생성된다.
        cat
            - 파일의 내용을 출력한다.
        cp
            - copy
            - 폴더나 파일을 복사한다.
                - 알고리즘적?으로 생각해보자.         
                  우리의 추상체계에 있는 복사는 말그대로 똑같은 것이 하나가 갑자기 띵동 생겨나는 것이다.    
                  하지만 컴퓨터 안에서는 파일을 임시 저장하고 하나를 생성하는 것이다. 그렇기에 파일의 크기가 크면 복사하는데 시간이 더 걸리는 것이다.
        rm
            - remove
            - 파일이나 폴더를 삭제한다.
                - 폴더를 삭제하려는 경우에는 속성을 주어야한다.
                    - r → recursion
                    - f → force 물어보지 않고 삭제한다.
        mv
            - move
            - 폴더나 파일의 이름을 변경한다.
                - mv helloworld helloworld2 → 이름이 helloworld2로 변경
            - 폴더나 파일의 위치를 변경한다.
                - mv helloworld ksj
            - 이것 또한 다른 관점에서 생각해보자.   
              우리의 추상체계에서 '옮긴다'는 물리적인 동작이다.      
              하지만 컴퓨터는 사람과 같은 물리적인 동작을 하지 않는다.      
              순서에 의해 실행할 뿐이다. 먼저 옮길 파일을 복사하여 임시저장하고, 원래의 파일을 지운 후, 도착폴더에서 파일을 생성한다.

    내가 찾아본 명령어
        man
            - manual을 뜻한다
            - man 명령어 → 명령어의 설명을 표기해준다.
            - q로 나간다.
        open
            - open . → 현재 directory를 gui로 보여준다.
            - 아직 마우스가 함께해야 편한 나에게는 여러모로 도움을 줄 것 같다.
        find
            - find 타입 이름
            - 전체영역에서 이름을 포함하는 모든 타입의 파일을 보여준다. (익숙해져보자)
        mkdir -p
            - 폴더를 하나만 만드는게 아니라 경로(트리를 만들수 있다.)
        echo
            - echo 입력내용 파일이름
                - 입력내용을 가진 파일을 생성한다.
        clear
            - 터미널을 깨끗하게 만들어준다.
        bye
            - 진행중인 모든 작업을 정리한다

    nano 사용
        nano 파일이름
        터미널 환경에서의 텍스트 에디터
            - visual studio나 intellij 같은 ide ???? ← 이건 팩트체크 해보자
            - 사용하는 이유 : 서버는 원격에서 조종해야 하는 일이 발생할 수 있다.

#### c. 관리자권한
    
- 새로운 프로그램을 설치하고 지울때는 관리자 권한으로 실행됨
        - 우리집에서는 내가 관리자이자 사용자이다. 물품들을 마음대로 사용하고, 이동하고, 또한 새로운 가구들을 구매하여 배치할 수 있다.
        - 한 가족이 한 집에 살게 되면 복잡해진다. 서로의 영역을 함부로 건들지 않도록 하는 관리규칙과 허가가 (= 관리자가 하는 일) 필요하다.
        - 타인이 운영하는 카페에 가면 나는 철저한 사용자이다. 주인장(관리자)의 허가 없이 함부러 새로운 가구를 비치할 수 없다. 만약 단골이 되어 이쁜 꽃을 선물한다면 관리자에게 권한을 얻어 배치 하도록 하자.
    - **절대 경로와 상대 경로**
        - 절대주소
            - pwd로 확인 가능한 경로 = 기준점으로 부터의 절대적인 위치
                - 기준점 (/ →루트폴더) 맥에서는 User
                    - 관리자의 영역
                    - sudo를 이용하여 관리자의 권한 획득 가능
        - 상대주소
            - 현재위치에서의 상대주소
                - . → 현재주소
                - .. → 현재주소의 상위주소
        - 나에게 대입하여 생각해보자. 나의 집은 00 (house폴더) 이다. 내가 밖에 나가 돈다고 해도 절대 변하지 않으며 그 주소를 누군가에게 알려줄때는 우리집은 00이야 (users/username/house)야 라고 말할 것이다. 하지만 내가 있는 위치는?? 매번 변한다. 내가 있는 위치(작업하고 있는 위치)는  . 으로 표현 가능한 것이다.

### 2. 패키지와 패키지 관리자

    - 리눅스에서도 프로그램을 깔아야 하지 않겠는가
    - 패키지는 하나의 프로그램을 구동하기위한 directory의 압축파일이라 생각하자
        - 하나의 패키지에는 다음과 같은 것들이 들어있다.
            - 프로그램 파일
            - 프로그램 설치파일
            - 프로그램 설치 설명서
            - 프로그램에 대한 정보를 담은 파일
    - 이러한 패키지들의 관리를 편하게 해주는 것이 패키지 매니저이다.
        - homebrew → 아이폰의 appstore 생각하자.

<br>


#### a. command line tools
    - 커맨드 라인 도구 설치 (command line tools)
        - xcode-select —install
    - Homebrew 패키지 매니저 설치
        - homebrew 사이트의 스크립트 복사 후 터미널에 복붙
        - Installation successful! 이 보이면 설치 완료
        - *설치과정에서의 시행착오*
            - *문제의 이유*
                - *터미널을 처음 만지다 보니 설치과정에서 멈춰있는 것을 버그로 생각해 도중에 꺼버렸다.*
            - *중간 과정*
                - *control c 를 눌러 종료 후 다시 스크립트를 복붙하여 설치를 진행하였다.*
            - *문제 발생*
                - *brew —version 에서는 버전이 뜬다. 하지만 추후 brew install wget이 작동하지 않는다.*
                - *오류메세지*
                    - *No available formula or cask with the name*
            - *해결*
                - *구글링*
                - *$ rm -rf $(brew —repo homebrew/core)*
                    - *repo → repository (저장소)*
                        - *homebrew의 패키지 파일 저장소인듯하다.*
                        - [https://www.lesstif.com/sonatype-nexus/software-repository-36209229.html](https://www.lesstif.com/sonatype-nexus/software-repository-36209229.html)  → 정독해보자 추후에도 도움이 될거같다. why? maven, gradle에 대한 설명
                    - *homebrew의 core directory 삭제*
            - *재설치*
            - *결과 - 정상작동*
            - *해결과정을 따른 이유*
                - *문제의 원인이 설치 도중 종료한 것이었다. 그렇다면 프로그램 파일들이 복사되다가 도중에 중단되었다고 유추하였고, 이를 해결하는 방법은 아예 백지상태로 돌리는 것이라고 생각했다.*
            - *앞으로 다른 비슷한 문제가 발생하면 core directory를 삭제하겠다.*

### 3. 환경변수

- 환경변수
    - 특정 키워드가 어떠한 역할 혹은 경로를 저장하도록 만들어 줌
    - 지역변수와 전역변수
    - "프로세스가 컴퓨터에서 동작하는 방식에 영향을 미치는, 동적인 값들의 모임이다."                    -위키백과-


- 지역 환경변수
    - 지정된 지역에서만 쓰이는 환경변수
    - ksj=helloworld
- 전역 환경변수
    - export ksj=helloworld
    - 변수에 공백이 존재할 경우 값 전체를 따옴표로 감싼다.
- env → 모든 환경변수를 볼 수 있다.
- **이부분은 추가공부가 필요하다. 어떤 의미인지 이해는 했다. 하지만 더 정확히 알아두어야 하는 중요한 부분같다.**
