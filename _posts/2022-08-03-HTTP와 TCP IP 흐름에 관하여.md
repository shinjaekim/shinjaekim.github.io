### 정리
내가 헷갈려서 정리부터 하고 들어가야겠다.

***1. HTTP는 무엇인가?***

    Hyper Text Transfer Protocol
        '하이퍼텍스트' 방식으로 데이터를 주고 받는 규칙

일단 하이퍼텍스트라는 방식부터 알아보자.

우리가 도서관에 가서 책을 읽으면 일반적으로 처음부터 끝까지 순서대로 읽는다. 그러니까 앞에서 뒤로 이동하는 방식으로 책을 읽게된다. 하지만 하이퍼텍스트는 ***앞->뒤의 개념이 아니다.*** 웹서핑을 하다가 볼수 있는 링크들을 누르면 다른 페이지로 이동하게 된다. 이렇게 ***중간->다른곳***의 개념으로 읽는 것이 하이퍼 텍스트이다.

그리고 HTTP는 이러한 데이터를 주고받는데 필요한 규칙이다. 따라서 이전에 정리했던 http messages는 이 규칙에 대한 설명이 되는 것이다.

그렇다면

<BR>

***2. 네트워크는 무엇이었는가??***
    
    컴퓨터와 컴퓨터를 연결하는 것

그리고 네트워크의 연결의 방법론을 tcp/ip 4계층을 통하여 간단하게 볼 수 있다.

    1. 응용계층에서 브라우저를 통해 데이터들을 요청한다.
        HTTP,FTP,Telnet,SMTP 등 네트워크를 사용하는 응용프로그램들

    2. 전송계층에서 tcp,udp등의 방법으로 송신자와 수신자를 연결한다
        TCP는 연결이 확인되면 전송하고, UDP는 일방적으로 전송한다.

    3. 인터넷계층에서 목적지까지의 ip경로를 찾는다.

    4. 네트워크연결계층에서 mac주소를 이용하여 네트워크와 연결한다.

<BR>

***3. 이 둘을 하나의 흐름으로 보면??***

    브라우져에서 도메인을 이용하여 특정 주소로 진입하는 상황을 가정하자.

첫째로, DNS상에서 ip주소를 받아와야한다.
1. 클라이언트가 리졸버에게 ip주소를 요청한다.
2. 리졸버가 root네임서버에 요청한다. -> TLD네임서버 주소를 알려준다.
3. 리졸버가 TLD네임서버에 요청한다. -> 권한있는네임서버 주소를 알려준다.
4. 리졸버가 권한있는네임서버에 요청한다 -> ip주소를 알려준다.
5. 해당 ip주소로 해당 웹서버에 접근한다.

둘째로, HTTP메시지를 작성한다.
- 해당 웹서버에서 찾고자 하는 데이터에 관련한 명령들을 해주는 것이다.

셋째로, 전송계층에서 TCP HTTP메시지를 패킷으로 분해한다.

넷째로, 인터넷 계층에서 ip경로를 확인하고

다섯째로, 네트워크 연결계층에서 mac주소를 확인하여 연결, 전송한다.

<BR>

***4. 여기에 웹 구성 개념까지 얹어보자***    
위의 과정을 통한다면 클라이언트와 서버가 연결되고 데이터가 전송된다. 그러면 서버에서 데이틑 어떻게 처리되는가??

    서버는 크게
        presentation layer
        application layer
        data access layer
    의 계층으로 나눌 수 있다.
    
여기에 부가적으로 세가지의 계층에 공통적으로 적용되어야 할 cross cutting과
외부 소프트웨어와의 연결을 하게되는 ***외부 API - third party integrations -***가 있다.
