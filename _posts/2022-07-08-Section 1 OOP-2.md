## 간단정리

- 생성자는 멤버는 아니다.
- 클래스로 만든 인스턴스를 초기화 할때 쓰인다.
    - 기본생성자와 매개변수를 받는 생성자가 있다.
    - 매개변수를 받는 경우 생성과 동시에 초기화가 가능하다.
    - 기본생성자는 클래스 안에 작성하지 않아도 컴파일러가 자동으로 생성해주고 있었다.
- 이너클래스는 멤버에 속한다 (멤버 - 필드 메서드 이너클래스)
    - 정적 내부 클래스는 static변수만 참조가능하다.
    - 캡슐화를 가능하게한다.
    - 클래스 안의 클래스이다.

## 정리


### 1. 생성자(constructor)

    - 인스턴스를 초기화 하는 메서드
    - 반환값이 없다. void 작성하는 메서드와는 다른 점
    - 멤버가 아니다. (멤버는 필드, 메서드, 이너클래스)
    - 매개변수를 가진 생성자와 그렇지 않은 생성자가 있다.
    - 매개변수를 가진 생성자는 호출과 동시에 리터럴 할당이 가능하다.

<br>

#### a. 생성자 

앞서 공부한 바를 떠올려보자. 클래스를 만들면 사용시 인스턴스로 객체화 한다. 식으로 본다면
 
    ‘생성한 클래스' 인스턴스 이름 = new 클래스이름() 
 
이 될 것이다. 그리고 new 키워드는 인스턴스를 생성하는 역할을 한다. **여기서!!** **new 다음의 클래스이름이 바로 생성자**로, **생성된 인스턴스를 초기화** 하는 역할을 한다. 초기화에 대해 조금 더 알고있는 내용들을 정리하며 파고들어보자. 일반적으로 변수를 선언할때, 예를 들어 num 이름을 가진 정수형 변수를 선언하면 식은 이렇게 될 것이다.
 
    int num → 할당까지 한 줄에 하면 int num = 0;

위의 사례처럼 0으로 선언한 변수를 초기화 한다. 즉, **num이라는 int형 변수를 0으로 초기화 해주었듯  클래스로 생성한 인스턴스에 할당, 초기화를 해주는 역할을 하는 메서드를 생성자** 라고 한다.


 

###### ㄱ. 리턴값이 없다.
- 메서드에서 리턴값이 없다면 void를 사용했지만, 생성자는 안써도 된다. 이유는? 그냥 생성자 자체가 리턴값이 없기 때문이다.
    
###### ㄴ. 생성자 하나는 필수.
- 안써도 진행 됐던 이유는 자바 컴파일러가 자동으로 만들어줬기 때문
    - 매개변수가 없는 기본 생성자를 만들어준다.

###### ㄷ. 왜 쓰는가??

```java
public class Main[
    public static void main(String[] args) {
        Man firstMan = new Man(); // 2. 인스턴스 생성시 이렇게 될 것이다. 이 때 인스턴스의 
                                 //    값들에 초기화를 진행하면 
        
        firstMan.height = 183.0;  // 3. 일일이 하나씩 해줘야한다. 4번으로 가보자.
        firstMan.name = "홍길동"

        Man secondMan = new Man(183.0 , "홍길동") // 5. 생성과 동시에 초기화가 가능하다.
    }
}

class Man{             // 1. 생성자 없이 필드값만 있는 경우를 가정해보자. 그렇다면 이 클래스
                        //    로 인스턴스를 생성하면 생성자를 따로 적지 않았기 때문에 컴파일러
    double height;       //    가 자동으로 기본생성자를 만들어 줄것이다. 2번으로가자.
    String name;

    Man(double height, String name){. // 4. 하지만 이렇게 매개변수를 가진 생성자를 생성하면
        this.height = height;           //    5번으로가보자
        this.name = name
    }
}
```
    

###### ㄹ. 클래스 내부 상호 호출
- this()
    - **클래스 내부에서 생성자를 호출**할 때
    - 생성자 내부에서만 사용 가능하며
    - 반드시 첫 줄에 쓰자
- this
    - **인스턴스 자신**을 가리킨다.
    - 매개변수가 있는 생성자의 매개변수 이름을 인스턴스 변수와 이름을 같게 했을 경우.
        - Class의 인스턴스변수와 구분하기 위해 인스턴스 변수는
            - this.인스턴스변수 로 작성한다.
